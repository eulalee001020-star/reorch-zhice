# 实验室 Replay 与计划员采纳证据

## 口径

这份数据用于说明当前项目如何记录“采纳 / 驳回 / 失败归因”，但不能写成客户生产现场采纳率。

当前阶段更准确的表述是：

- 数据来源：受控实验室 replay / 数字孪生样本 / 内部计划员式评审。
- 可证明：系统能记录推荐、采纳、微调、驳回和原因归因。
- 不能证明：客户现场一定达到同样 ROI、采纳率或延期改善。

## 初始样本统计

| 指标 | 当前样本 |
| --- | --- |
| replay case 数 | 12 |
| Top-1 直接采纳 | 7 |
| Top-K 内微调后采纳 | 3 |
| 驳回 / 退回人工 | 2 |
| 严格 Top-1 采纳率 | 58.3% |
| Top-K 覆盖采纳率 | 83.3% |
| 驳回率 | 16.7% |
| 数据问题导致不可计入 ROI | 2 |
| 策略 / 权重偏好导致微调 | 3 |

采纳率口径：

```text
strict_adoption_rate = Top-1 直接采纳 / 全部 replay cases = 7 / 12 = 58.3%
topk_covered_adoption_rate = (Top-1 采纳 + Top-K 内微调采纳) / 全部 replay cases = 10 / 12 = 83.3%
```

## 结论

这组数据的价值不在于证明“客户现场已经成功”，而在于证明产品闭环开始成立：系统能记录为什么采纳、为什么微调、为什么退回人工，以及哪些数据问题不能计入 ROI。

下一阶段应把这张表替换为客户只读数据的历史 replay，并至少累计 10-30 条真实历史异常。
